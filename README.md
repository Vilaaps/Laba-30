### 2. Таблица атрибутов и настроек JSON

| Атрибут / настройка              | Что делает              | Пример                                  |
|----------------------------------|-------------------------|-----------------------------------------|
| `[JsonPropertyName("name")]`     | Задаёт имя поля в JSON  | `"name"` вместо `"Name"`                |
| `[JsonIgnore]`                   | Поле не попадает в JSON | Пароли, внутренние поля (InternalNotes) |
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase    | `"realName"` вместо `"RealName"`        |
| `JsonStringEnumConverter`        | Enum как строка         | `"Marvel"` вместо `0`                   |
| `WriteIndented = true`           | JSON с отступами        | Для удобства при разработке             |

### 3. Главные выводы

1. **Сериализация -** это превращение объекта из памяти программы в текстовый формат (JSON), чтобы его можно было передать по сети. Десериализация — это когда мы берем JSON-текст и превращаем его обратно в объект.

2. **[JsonIgnore]** нужен для безопасности и чистоты данных. Например, в классе `User` у нас может быть поле `PasswordHash` или `AdminNotes` . Мы помечаем их `[JsonIgnore]`, чтобы эти данные никогда не ушли клиенту в браузере.

3. **Marvel** гораздо легче читать и отлаживать, чем цифры `0`, `1`. Кроме того, если мы добавим новый элемент в середину Enum в коде, числовые значения у существующих элементов могут сдвинуться, что сломает работу фронтенда. Текстовые имена более стабильны.

4. По умолчанию поля в JSON начинались с заглавной буквы . После включения **CamelCase** все ключи стали начинаться со строчной буквы `realName`, что является стандартом в JavaScript и облегчает работу фронтенд-разработчикам.
